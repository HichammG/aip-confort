A set of REST APIs are released by the project to access both data that has been monitored over time in one or more of these pilots, along with control commands that result from the designed AIP Confort algorithm

Access Granting

To be authorized to use the AIP Confort REST APIs, you need to have an university email account that contain :

  • @univ-lorraine.fr
  • @admin.univ-lorraine.fr

POST Signup

If you haven't an account, you need to create one with the following link : 

            
              https://aip-confort.milebits.com:3001/signup

Body of the request shall contain the following data : 

              
                {
                    "email" : xxx@univ-lorraine.fr
                    "password" : xxx
                    "username" : xxx
                }
Body Form
email required User email from university
password required password that will be use to Login
username required Username that will be display in the website

Possible response codes:

Code Payload (application/json) Meaning
200 
                  
                    {
                      "message": "user created successfully",
	                  "status": "successful"
                    }
Success
401 { "message": "user with that email exists", "status": "fail" } Error: User already exist
401 { "message": "user denied", "status": "fail" } Error: Wrong User email
500 { "message": "error", "status": "fail" } Error: Database problem

Parameters
token required Client token generated using JWT (for further information on JWB Website). You can find your token in the website header : Click here to go to the website

Code Payload (application/json) Meaning
200 
                    
                      {
                      "data": [
                        {
                            "S108": [
                                {
                                    "CO2": true,
                                    "Humidite": true,
                                    "Luminosite": true,
                                    "Son": true,

POST Add microcontrôleur

If you are administrator you can add microcontrôleur in the architecture and it should be done using the following GET request:

              
                https://aip-confort.milebits.com:3001/ajouterCapteurs?token={token}
Parameters
token required Client token generated using JWT (for further information on JWB Website). You can find your token in the website header : Click here to go to the website

Body of the request shall contain the following data : 

              
                {
                    "ip":"193.50.135.205",
                    "Salle":"S108",
                    "Temperature":true,
                    "Acoustique":true,
                    "Luminosite":true,
                    "CO2":true,
                    "Humidite":true
                }
Body FORM
ip required IP address of the microcontrôleur
Salle required Room where the microcontrôleur is
Temperature required Starting state of the temperature sensor (it will be change by the microcontrôleur automatically)
Humidite required Starting state of the humidity sensor (it will be change by the microcontrôleur automatically)
Acoustique required Starting state of the sound sensor (it will be change by the microcontrôleur automatically)
CO2 required Starting state of the co2 sensor (it will be change by the microcontrôleur automatically)
Luminosite required Starting state of the luminosity sensor (it will be change by the microcontrôleur automatically)

Possible response codes:

Code Payload Meaning
200 
                      {
                                "data": "Done",
                                "message": "Capteurs modified",
                                "status": "successful"
                            }
Success
401 {"message": "Already exist"} Error: microcontrôleur with this ip already exist
401 {"message": "wrong token"} Error: Wrong token
401 {"message": "any token"} Error: Token missing in URL
500 {"message": An error} Error: Internal error
